# gstack 스킬 가이드

> gstack은 Claude Code를 전문 역할별 에이전트로 구조화하는 스킬 팩입니다.
> 각 스킬은 `/스킬명`으로 호출하거나, 파이프라인에서 자동으로 실행됩니다.

## 전체 파이프라인 (기능 개발 시)

```
요청 → /office-hours → /plan-ceo-review → /plan-eng-review
     → /design-consultation → /autoplan → BUILD
     → /design-review → /review → /qa → /cso
     → /ship → /document-release → /retro → /checkpoint
```

---

## 스킬 목록 (카테고리별)

### 기획 · 전략

| 스킬 | 설명 |
|------|------|
| `/office-hours` | **YC 오피스 아워**. 스타트업 모드: 수요 검증 6가지 질문. 빌더 모드: 사이드 프로젝트 브레인스토밍 |
| `/plan-ceo-review` | **CEO/창업자 관점 리뷰**. 문제를 재정의하고, 10점짜리 제품을 찾고, 전제를 도전 |
| `/autoplan` | **자동 파이프라인**. CEO, 디자인, 엔지니어링 리뷰를 순차 실행. 자동/수동 결정 혼합 |

### 아키텍처 · 엔지니어링

| 스킬 | 설명 |
|------|------|
| `/plan-eng-review` | **엔지니어링 매니저 리뷰**. 아키텍처, 데이터 흐름, 엣지케이스, 테스트 커버리지, 성능 점검 |
| `/investigate` | **체계적 디버깅**. 조사 → 분석 → 가설 → 구현 4단계. 원칙: 원인 없이 수정 금지 |
| `/health` | **코드 품질 대시보드**. 타입체커, 린터, 테스트, 데드코드 감지 종합 점수 |

### 디자인

| 스킬 | 설명 |
|------|------|
| `/design-consultation` | **디자인 시스템 설계**. 제품 이해 → 경쟁사 리서치 → 디자인 시스템 제안 → `DESIGN.md` 생성 |
| `/design-shotgun` | **디자인 변형 탐색**. 여러 AI 디자인 시안 생성 & 비교 보드 피드백 |
| `/design-html` | **프로덕션 HTML/CSS 생성**. 승인된 목업을 실제 코드로 변환 |
| `/plan-design-review` | **디자이너 관점 플랜 리뷰**. 각 디자인 차원 0-10 평가 |
| `/design-review` | **비주얼 QA**. 시각적 불일치, 간격, 계층 구조, AI 슬롭 패턴 검출 & 수정 |

### 테스트 · QA

| 스킬 | 설명 |
|------|------|
| `/qa` | **브라우저 QA + 버그 수정**. 실제 테스트 → 버그 수정 → 원자적 커밋 → 재검증 |
| `/qa-only` | **QA 리포트만**. 수정 없이 버그 리포트, 헬스 스코어, 스크린샷, 재현 단계만 제공 |
| `/benchmark` | **성능 회귀 감지**. 페이지 로드 시간, Core Web Vitals, 리소스 크기 비교 |
| `/canary` | **배포 후 카나리 모니터링**. 콘솔 에러, 성능 회귀, 페이지 실패 감시 |

### 보안

| 스킬 | 설명 |

## 사용 팁

1. **기능 개발**: "~만들고 싶어" → 전체 파이프라인 자동 실행
2. **버그 수정**: "~가 안 돼" → `/investigate` 직접 라우팅
3. **빠른 수정**: "빠르게 ~해줘" → BUILD → `/review` → `/qa` → `/ship`
4. **안전한 작업**: `/guard`를 먼저 실행하면 파괴적 명령 + 편집 범위 제한
5. **디자인 탐색**: `/design-shotgun`으로 시안 비교 → `/design-consultation`으로 확정
