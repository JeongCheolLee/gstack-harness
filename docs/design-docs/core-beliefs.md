# Core Beliefs — 핵심 신념

## 1. 에이전트 우선 (Agent First)

코드를 직접 작성하지 않는다. 에이전트가 코드를 작성하고, 사람은 환경을 설계한다.
에이전트가 어려움을 겪으면 "더 분발하라"가 아니라 "무엇이 누락되었는가?"를 질문한다.

## 2. 리포지터리가 기록 시스템 (Repo as System of Record)

모든 지식은 리포지터리 내 버전관리되는 아티팩트에 존재해야 한다.
Google Docs, Slack, 사람의 머릿속에만 있는 지식은 시스템에서 접근 불가.

## 3. 제약이 곧 속도 (Constraints Enable Speed)

아키텍처 경계와 린트 규칙이 에이전트의 자율성을 보장한다.
제약 없는 자유는 드리프트를 낳고, 드리프트는 속도를 죽인다.

## 4. 수정 비용 < 대기 비용 (Fix > Wait)

완벽을 기다리지 않는다. 빠르게 병합하고 후속 PR로 개선한다.
에이전트 처리량이 사람의 주의력을 초과하는 시스템에서 대기는 가장 비싼 선택.

## 5. 가비지 컬렉션은 지속적으로 (Continuous GC)

기술 부채는 고금리 대출. 매일 조금씩 갚는 것이 한꺼번에 갚는 것보다 낫다.
"황금 원칙" 편차를 정기적으로 검사하고, 정리 PR을 자동 생성한다.

## 6. 점진적 공개 (Progressive Disclosure)

에이전트에게 1,000페이지의 설명서가 아닌 맵을 제공한다.
작은 진입점에서 시작하여 필요할 때 심층 정보를 탐색한다.
