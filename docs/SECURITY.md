# 보안 원칙 (Security Principles)

## 경계에서 검증

- 외부 입력(사용자 입력, API 응답)은 경계에서 파싱 & 검증
- 내부 코드 간에는 타입 시스템을 신뢰

## 시크릿 관리

- `.env` 파일은 절대 커밋하지 않음 (.gitignore에 포함)
- 시크릿은 환경 변수로만 주입
- `.env.example`에는 실제 값 대신 플레이스홀더만 기재

## 의존성

- 리포지터리 내에서 추론 가능한 의존성 선호
- 외부 라이브러리 추가 시 보안 영향 검토

---

*제품이 결정되면 인증, 인가, 데이터 보호 등 구체적인 보안 정책을 추가한다.*
