## 레이어 아키텍처

각 비즈니스 도메인은 아래 레이어 순서를 따른다:

```
┌─────────────────────────────────┐
│            UI / Routes          │  ← 사용자 접점
├─────────────────────────────────┤
│            Services             │  ← 비즈니스 로직
├─────────────────────────────────┤
│          Repositories           │  ← 데이터 접근
├─────────────────────────────────┤
│            Models               │  ← 데이터 모델
├─────────────────────────────────┤
│         Config / Types          │  ← 설정 & 타입 정의
└─────────────────────────────────┘

교차 관심사 ──→ [Providers] ──→ Auth, Telemetry, Feature Flags
```

### 의존성 규칙
- 위 → 아래 방향만 허용
- 같은 레이어 간 의존성 금지
- 교차 관심사는 반드시 Providers 인터페이스를 통해 주입

## 에이전트 피드백 루프

```
사람 (프롬프트)
    ↓
에이전트 (코드 생성)
    ↓
자동 검증 (린터, 테스트, CI)
    ↓
에이전트 리뷰 (자체 PR 리뷰)
    ↓
사람 검토 (필요시)
    ↓
병합
    ↓
가비지 컬렉션 (정기 정리 PR)
```
